# AI Universe Lite — Stage 7.5 Deploy

**Deterministic.** Cloud Run deploy via the same `deploy.sh` contract as `jleechanorg/ai_universe`.

## Behavior

```bash
gcloud builds submit \
  --config gems/<id>/cloudbuild.gem.yaml \
  --substitutions=_GEM_ID=<id>,_GEM_VERSION=<version>,_REGISTRY=gcr.io/ai-universe-2025

gcloud run deploy <service-name> \
  --image gcr.io/ai-universe-2025/gem-<id>:<version> \
  --region us-central1 \
  --service-account <default-compute-sa>@ai-universe-2025.iam \
  --allow-unauthenticated \
  --port 8080 \
  --cpu 1 --memory 512Mi --timeout 300 \
  --min-instances 0 --max-instances 10 --concurrency 80 \
  --set-env-vars "NODE_ENV=production,PORT=8080,MCP_SESSION_STORE=memory,GEM_ID=<id>,GEM_VERSION=<version>,REF_BUCKET=ai-universe-lite-refs,FIREBASE_PROJECT_ID=ai-universe-b3551,MCP_SERVER_PORT=8080,STORAGE_TYPE=firestore,FIRESTORE_PROJECT_ID=ai-universe-b3551" \
  --set-secrets "OPENAI_API_KEY=OPENAI_API_KEY:latest,ANTHROPIC_API_KEY=ANTHROPIC_API_KEY:latest,..." \
  --labels "gem-id=<id>,gem-version=<version>,gem-builder=true"
```

Service name pattern: `gem-<id>-<env>` where `<env>` ∈ `dev` | `staging` | `prod`.

## Prod guard (mirrored from `ai_universe/deploy.sh` lines 162-193)

```bash
if [[ "$ENVIRONMENT" == "prod" ]] && [[ "${GITHUB_ACTIONS:-false}" != "true" ]]; then
  echo "❌ PRODUCTION GEM DEPLOY BLOCKED"
  echo "Use: https://github.com/jleechanorg/ai_universe_lite/actions/workflows/gem-publish.yml"
  exit 1
fi
```

Local `prod` deploys are **always blocked**. The override is `ALLOW_LOCAL_PROD_DEPLOY=true` in `.env.local` (Jeffrey-only, never committed).

## PR previews

When a gem PR is opened, `gem-preview-pr.yml` (Phase 1) auto-deploys a `gem-preview-pr-<PR_NUMBER>-<gemId>` service with 6h TTL + 1h idle shutoff.

## Why deterministic

Deploy is a script invocation. We want it reproducible and auditable.
